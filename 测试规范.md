# 数据准备
- 40万条房源数据
- 每条房源5个字段5次修改

# 数据库命名规范
- 以测试方案对应序号开始，如1_house
- 模拟200个字段名字如column1-column200命名
- column1=houseid，column10=电话，column20=创建人，column30=创建时间，再有特殊字段另做约定。

# 测试顺序
- 按照测试方案的2、6、1、3、4、5、7的顺序

# 数据库 PressureTest

# 2号方案 json格式
- 数据量：40万
- 只有5个字段有修改记录（4-8）
- 数据表：t2_house
# 6号方案 0°
  - 相当于所有字段都修改过。每个字段修改5次
- -数据量：400000*5+400000*5*4=1000万 
- 数据表：t6_house, t6_tel  
# 1号方案 表转90°
- 每个字段增加一次，4-8个字段修改5次
- 数据量：400000*200+400000*5*4=8800万
- 数据表： t1_code字段字典表， t1_house房源表，t1_history修改历史记录表
# 3号方案 90°+json
- 数据量：40万*200=800万
- 数据表： t1_code字段字典表， t1_house房源表，t3_json修改历史记录表
# 4号方案 0°+json
- 数据量：40万*5=200万
- 数据表：t4_house，t4_tel,t4_code,t4_json
# 5号方案 0°+90°
- 数据表：t4_house, t4_tel，t4_code，t5_history
- 数据量：40万*5*5=1000万
